# ü§ù Contributing to the Solana KYC Compliance SDK

We welcome contributions from the community! By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## Getting Started

1.  **Fork** the repository on GitHub.
2.  **Clone** your forked repository:
    ```bash
    git clone [https://github.com/YOUR_USERNAME/solana-kyc-compliance-sdk.git](https://github.com/YOUR_USERNAME/solana-kyc-compliance-sdk.git)
    ```
3.  Install dependencies:
    ```bash
    npm install # Or your package manager equivalent
    ```

## üêõ Found a Bug?

If you find a bug, please check the [Issues](https://github.com/Gitdigital-products/solana-kyc-compliance-sdk/issues) page to see if it has already been reported. If not, please submit a new issue with:
* A clear and descriptive title.
* Steps to reproduce the bug.
* Expected and actual behavior.
* The version of the SDK you are using.

## ‚ú® Suggesting Enhancements or Features

We love new ideas! Before submitting a Pull Request for a major feature, please open an issue first to discuss your idea with the maintainers. This prevents duplicate work and ensures the feature aligns with the project's goals.

## Submitting a Pull Request (PR)

1.  Create a new branch for your changes:
    ```bash
    git checkout -b feature/your-feature-name
    ```
2.  Make your changes, ensuring code follows the project's style.
3.  Add tests for any new functionality or bug fixes.
4.  Ensure all existing tests pass.
    ```bash
    npm test
    ```
5.  Commit your changes with a descriptive commit message (e.g., `feat: added support for metadata lookups`).
6.  Push your branch to your forked repository.
7.  Open a Pull Request from your branch to the **`main`** branch of the original repository.

Thank you for contributing!
